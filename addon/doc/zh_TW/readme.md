# NVDA 螢幕簾幕密碼

這個 NVDA 附加元件會在「螢幕簾幕」(Screen Curtain) 的隱私操作前加入密碼保護。

主要功能：

- 停用螢幕簾幕前要求輸入密碼。
- 螢幕簾幕作用中時，結束或重新啟動 NVDA 前要求輸入密碼。
- 僅儲存加鹽的 PBKDF2-HMAC-SHA256 密碼雜湊值，不會儲存明文密碼。
- 密碼欄位預設會隱藏內容，旁邊提供「顯示密碼」核取方塊。
- 忘記密碼時，可在密碼提示對話方塊中輸入 `00000` 並等待五分鐘以重設密碼。

可從 NVDA 功能表的「偏好」>「設定」>「螢幕簾幕密碼」開啟設定畫面。

## 注意事項與已知行為

- 需要 NVDA 2026.1 或更新版本(此附加元件掛接的是該版本才新增的隱私權與安全性設定面板)。
- 只有使用者主動觸發的真正 NVDA **結束** 動作才會受到密碼保護。重新啟動 NVDA，以及因更新或安裝附加元件而觸發的結束動作，皆刻意設計為 **不會** 被阻擋，避免中斷這些流程。
- 取消受保護的結束動作會讓 NVDA 繼續執行；此時 NVDA 的記錄檔可能會寫入一行無害的「logic error」訊息。
- 保護範圍涵蓋使用者可觸及的停用途徑：切換手勢(NVDA+control+escape)，以及隱私權與安全性設定中的螢幕簾幕核取方塊。若程式碼(例如其他附加元件)直接呼叫螢幕簾幕控制器的 `disable()`，則不會被攔截。

本附加元件的靈感來自 nvaccess/nvda 議題 #20335。
